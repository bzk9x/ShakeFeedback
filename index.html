Features

Everything you need,
nothing you don't.

Built to integrate in minutes with sensible defaults tuned from real device data.

Four-Layer Architecture
Hardware acquisition, signal processing, orchestration, and presentation — cleanly separated so you only touch what you need.
Auto Screenshot Capture
Captures the current Activity screen asynchronously, non-blocking, at the exact moment of shake. Can be disabled for privacy-sensitive apps.
Haptic Profiles
Three built-in patterns: LIGHT_CLICK, DOUBLE_TAP, and HEAVY_THUD. Or define custom timing and amplitude arrays.
Lifecycle Aware
Sensor binds and unbinds automatically with Activity or LifecycleOwner. Zero battery drain in background — no wake locks, ever.
Smart Signal Processing
Euclidean norm + gravity isolation + exponential moving average + debounce. Filters walking, running, and vehicle vibration with no false positives.
Diagnostic Collection
Automatically captures device model, OS version, memory stats, storage, and last 50 logcat lines. No network calls — local only.
Architecture

Four layers.
Zero coupling.

Each concern lives in exactly one layer. Swap, extend, or test any layer independently.

1
Hardware Acquisition
AccelerometerDataSource
Raw sensor polling at 60Hz. Pure hardware abstraction — no decisions, just (x,y,z) vectors.
2
Signal Processing
ShakeDetector
Mathematical pipeline: magnitude → gravity isolation → noise reduction → threshold crossing → debounce.
3
Orchestration
ShakeFeedbackManager
Observes Activity lifecycle, manages sensor binding, triggers haptics, and captures screenshots and diagnostics.
4
Presentation
ShakeFeedbackCallback
Pure callback interface to your app. Implement custom UI or use the built-in BottomSheet — your choice.
Quick Start

Up and running
in 5 minutes.

Three setup paths depending on how much you want to customize.

Basic Setup
Custom Config
Built-in UI
// In your Activity onCreate()
ShakeFeedback.initialize(
    activity = this,
    callback = object : ShakeFeedbackCallback {

        override fun onShakeDetected(event: ShakeEvent) {
            showFeedbackForm(event)
        }

        override fun onFeedbackSubmitted(data: FeedbackData) {
            sendFeedbackToServer(data)
        }

        override fun onFeedbackDismissed() { }

        override fun onError(message: String) { }
    }
)
val config = ShakeFeedbackConfig.Builder()
    .shakeThreshold(12.0f)          // More sensitive
    .hapticPattern(HapticProfile.HEAVY_THUD)
    .autoCaptureScreenshot(true)
    .debounceInterval(2000L)        // Longer cooldown
    .smoothingFactor(0.9f)
    .minimumThresholdCrossings(2)
    .build()

ShakeFeedback.initialize(
    activity = this,
    config = config,
    callback = myCallback
)
override fun onShakeDetected(event: ShakeEvent) {

    val sheet = ShakeFeedbackBottomSheet.newInstance(
        screenshot    = event.screenshot,
        diagnosticLogs = event.diagnosticLogs
    )

    sheet.setOnFeedbackSubmittedListener { feedback ->
        uploadFeedback(feedback)   // Send to your server
    }

    sheet.show(supportFragmentManager, "feedback")
}

Zero background drain. Sensor is unregistered the moment the app pauses. No wake locks, no idle polling.
Constant memory footprint. No accumulating caches — memory usage stays flat regardless of how many shakes occur.
No network calls. All processing is local. No tracking, no external pings — your data stays on device.
Graceful degradation. Works on devices without a vibrator. Haptic feedback silently skipped when unavailable.
Privacy-ready. Screenshot capture is opt-in. Disable globally for banking or healthcare apps with one config line.
